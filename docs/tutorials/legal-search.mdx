In this tutorial, you'll build a legal document search system that can analyze contracts, extract specific clauses, and answer complex legal questions using LLM reasoning.

## What You'll Build

A contract analysis system that can:
- Search across multiple contracts by keywords
- Filter by contract type, date, and parties
- Extract specific clauses (termination, liability, payment)
- Answer natural language questions about obligations

1. **Ingest contracts** - Store NDAs, service agreements, and employment contracts
2. **Build searchable index** - ReasonDB creates tree structures and search indexes
3. **Query with filters** - Filter by contract type, then search or reason
4. **Extract insights** - LLM extracts specific clauses and answers questions

## Prerequisites

- ReasonDB running locally
- An LLM API key configured
- Basic familiarity with RQL (see [RQL Tutorial](/tutorials/rql-basics))

## Step 1: Create the Legal Table

<Steps>
  <Step title="Create a table for contracts">
    ```bash
    curl -X POST http://localhost:4444/v1/tables \
      -H "Content-Type: application/json" \
      -d '{
        "name": "contracts",
        "description": "Legal contracts and agreements",
        "metadata": {
          "department": "legal",
          "retention_years": 7
        }
      }'
    ```

    Save the returned table ID.
  </Step>
</Steps>

## Step 3: Basic Contract Queries

### List All Contracts

```sql
SELECT title, metadata.contract_type, metadata.value_usd FROM contracts
```

### Filter by Contract Type

```sql
SELECT * FROM contracts WHERE metadata.contract_type = 'nda'
```

### Filter by Value

```sql
-- High-value contracts (> $100k)
SELECT title, metadata.value_usd FROM contracts
WHERE metadata.value_usd > 100000
```

## Step 4: Keyword Search

Search for specific terms across all contracts:

```sql
-- Find contracts mentioning "termination"
SELECT * FROM contracts SEARCH 'termination notice'

-- Find contracts about "confidential information"
SELECT * FROM contracts SEARCH 'confidential information disclosure'
```

## Step 5: Intelligent Legal Questions

Now let's use REASON for complex legal questions:

### Extract Termination Clauses

```sql
SELECT * FROM contracts REASON 'What are the termination notice periods for each contract?'
```

### Understand Liability Limits

```sql
SELECT * FROM contracts
WHERE metadata.contract_type = 'msa'
REASON 'What are the liability limitations and caps?'
```

### Find Confidentiality Obligations

```sql
SELECT * FROM contracts
REASON 'What are our confidentiality obligations and how long do they last?'
```

### Compare Payment Terms

```sql
SELECT * FROM contracts
REASON 'What are the payment terms and what happens if payment is late?'
```

## Step 6: Advanced Analysis

### Contract Portfolio Summary

```sql
-- Count contracts by type
SELECT COUNT(*) AS count, metadata.contract_type
FROM contracts
GROUP BY metadata.contract_type
```

### Total Contract Value

```sql
SELECT SUM(metadata.value_usd) AS total_value FROM contracts
```

### Combined Filter + Reason

Find specific contract types and extract key terms:

```sql
SELECT * FROM contracts
WHERE tags CONTAINS 'active' AND metadata.value_usd > 0
REASON 'What are our key financial obligations?'
```

## Step 7: Building a Contract Review Workflow

Here's a complete workflow for reviewing a new contract:

```sql
-- 1. Check for similar existing contracts
SELECT title, metadata.contract_type FROM contracts
WHERE metadata.contract_type = 'msa'

-- 2. Search for relevant precedents
SELECT * FROM contracts SEARCH 'service level agreement uptime'

-- 3. Compare liability clauses
SELECT * FROM contracts
WHERE metadata.contract_type = 'msa'
REASON 'What liability caps and limitations have we agreed to in the past?'

-- 4. Check standard termination terms
SELECT * FROM contracts
REASON 'What termination provisions are standard in our contracts?'
```

## Best Practices

<Tip>
**Organize by contract type**: Use metadata.contract_type to quickly filter contracts before applying expensive REASON queries.
</Tip>

<Note>
**Tag active vs archived**: Use tags like "active" and "archived" to exclude old contracts from searches.
</Note>

<Warning>
**Confidentiality**: Ensure your ReasonDB instance is properly secured when storing sensitive legal documents.
</Warning>
