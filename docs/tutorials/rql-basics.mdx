In this tutorial, you'll master RQL (ReasonDB Query Language) by building queries step by step. By the end, you'll be able to write complex queries that combine filtering, full-text search, and LLM reasoning.

## What You'll Learn

- Basic SELECT queries
- Filtering with WHERE clauses
- Full-text search with SEARCH
- LLM reasoning with REASON
- Aggregations and GROUP BY
- Combining clauses for powerful queries

## Part 1: Basic SELECT

The simplest query selects all documents from a table:

```bash
curl -X POST http://localhost:4444/v1/query \
  -H "Content-Type: application/json" \
  -d '{"query": "SELECT * FROM books"}'
```

**Response:**
```json
{
  "documents": [
    {"id": "doc_1", "title": "The Quantum Garden", "author": "James Morrison", ...},
    {"id": "doc_2", "title": "The Silent Witness", "author": "Elizabeth Shaw", ...},
    {"id": "doc_3", "title": "Mars Colony One", "author": "James Morrison", ...}
  ],
  "total_count": 3,
  "execution_time_ms": 12
}
```

### Select Specific Fields

You can select only the fields you need:

```bash
curl -X POST http://localhost:4444/v1/query \
  -d '{"query": "SELECT title, author FROM books"}'
```

<Tip>
Selecting fewer fields can improve performance for large result sets.
</Tip>

## Part 2: Filtering with WHERE

### String Equality

Find books by a specific author:

```bash
curl -X POST http://localhost:4444/v1/query \
  -d '{"query": "SELECT * FROM books WHERE author = '\''James Morrison'\''"}'
```

**Result:** Returns "The Quantum Garden" and "Mars Colony One"

### Numeric Comparisons

Find highly-rated books:

```bash
curl -X POST http://localhost:4444/v1/query \
  -d '{"query": "SELECT title, metadata.rating FROM books WHERE metadata.rating > 4.3"}'
```

**Result:** Returns books with rating > 4.3

### Tag Filtering

Find books with a specific tag:

```bash
curl -X POST http://localhost:4444/v1/query \
  -d '{"query": "SELECT * FROM books WHERE tags CONTAINS '\''science-fiction'\''"}'
```

**Result:** Returns the two sci-fi books

### Combining Conditions

Use AND/OR for complex filters:

```bash
# Science fiction books by James Morrison
curl -X POST http://localhost:4444/v1/query \
  -d '{"query": "SELECT * FROM books WHERE author = '\''James Morrison'\'' AND tags CONTAINS '\''science-fiction'\''"}'
```

### Pattern Matching

Use LIKE for partial matches:

```bash
# Titles containing "The"
curl -X POST http://localhost:4444/v1/query \
  -d '{"query": "SELECT * FROM books WHERE title LIKE '\''The%'\''"}'
```

## Part 3: Full-Text Search with SEARCH

The SEARCH clause uses BM25 for fast keyword matching:

```bash
curl -X POST http://localhost:4444/v1/query \
  -d '{"query": "SELECT * FROM books SEARCH '\''quantum computing'\''"}'
```

**Response:**
```json
{
  "documents": [{
    "id": "doc_1",
    "title": "The Quantum Garden",
    "score": 8.5,
    "highlights": ["...way to grow <b>quantum</b> <b>computers</b> organically..."]
  }],
  "total_count": 1,
  "execution_time_ms": 23
}
```

### Combine SEARCH with WHERE

Filter first, then search:

```bash
curl -X POST http://localhost:4444/v1/query \
  -d '{"query": "SELECT * FROM books WHERE author = '\''James Morrison'\'' SEARCH '\''Mars survival'\''"}'
```

<Note>
SEARCH is fast (~50ms) because it uses a pre-built BM25 index. Use it for keyword lookups.
</Note>

## Part 4: LLM Reasoning with REASON

The REASON clause uses an LLM to understand your question and extract answers:

```bash
curl -X POST http://localhost:4444/v1/query \
  -d '{"query": "SELECT * FROM books REASON '\''What book deals with artificial consciousness?'\''"}'
```

**Response:**
```json
{
  "documents": [{
    "id": "doc_1",
    "title": "The Quantum Garden",
    "answer": "The Quantum Garden deals with artificial consciousness. Dr. Sarah Chen must decide whether to give her quantum AI consciousness, exploring the ethics of creating sentient machines.",
    "confidence": 0.91
  }],
  "total_count": 1,
  "execution_time_ms": 2340
}
```

### Complex Questions

REASON excels at understanding complex questions:

```bash
# Find books about isolation
curl -X POST http://localhost:4444/v1/query \
  -d '{"query": "SELECT * FROM books REASON '\''Which books explore themes of isolation or being cut off?'\''"}'
```

### Combine Everything

The most powerful queries combine WHERE, SEARCH, and REASON:

```bash
curl -X POST http://localhost:4444/v1/query \
  -d '{"query": "SELECT * FROM books WHERE tags CONTAINS '\''science-fiction'\'' REASON '\''What ethical dilemmas do the protagonists face?'\''"}'
```

## Part 5: Aggregations

### Count Documents

```bash
curl -X POST http://localhost:4444/v1/query \
  -d '{"query": "SELECT COUNT(*) FROM books"}'
```

### Group By

Count books per author:

```bash
curl -X POST http://localhost:4444/v1/query \
  -d '{"query": "SELECT COUNT(*) AS book_count, author FROM books GROUP BY author"}'
```

**Response:**
```json
{
  "aggregates": [
    {"name": "book_count", "value": 2, "group_key": [["author", "James Morrison"]]},
    {"name": "book_count", "value": 1, "group_key": [["author", "Elizabeth Shaw"]]}
  ]
}
```

### Average Rating

```bash
curl -X POST http://localhost:4444/v1/query \
  -d '{"query": "SELECT AVG(metadata.rating) AS avg_rating FROM books"}'
```

## Part 6: Sorting and Pagination

### Order Results

```bash
# Sort by rating descending
curl -X POST http://localhost:4444/v1/query \
  -d '{"query": "SELECT * FROM books ORDER BY metadata.rating DESC"}'
```

### Paginate Results

```bash
# First page (2 results)
curl -X POST http://localhost:4444/v1/query \
  -d '{"query": "SELECT * FROM books ORDER BY title LIMIT 2"}'

# Second page
curl -X POST http://localhost:4444/v1/query \
  -d '{"query": "SELECT * FROM books ORDER BY title LIMIT 2 OFFSET 2"}'
```

## Summary

| Clause | Purpose | Speed |
|--------|---------|-------|
| `SELECT` | Choose fields | Fast |
| `WHERE` | Filter by metadata | Fast |
| `SEARCH` | BM25 keyword search | Fast (~50ms) |
| `REASON` | LLM semantic search | Slower (~2-5s) |
| `GROUP BY` | Aggregate results | Fast |
| `ORDER BY` | Sort results | Fast |
| `LIMIT/OFFSET` | Paginate | Fast |
