In this tutorial, you'll build an intelligent knowledge base that can answer user questions by understanding intent and extracting precise answers from your documentation.

## What You'll Build

A Q&A system that can:
- Answer natural language questions about your product
- Find relevant documentation sections
- Provide accurate answers with confidence scores
- Handle variations in how questions are asked

1. **Ingest documentation** - Add guides, FAQs, and troubleshooting docs with category tags
2. **User asks question** - Natural language query like "How do I reset my password?"
3. **Filter and reason** - Optionally filter by category, then LLM finds the answer
4. **Return with confidence** - Answer includes confidence score and source document

## Step 4: Scoped Queries

Filter to specific documentation areas:

### Only Search Getting Started

```sql
SELECT * FROM knowledge_base
WHERE tags CONTAINS 'getting-started'
REASON 'How do I invite team members?'
```

### Only Search Troubleshooting

```sql
SELECT * FROM knowledge_base
WHERE metadata.category = 'troubleshooting'
REASON 'Why is the app slow?'
```

### Only Search Billing

```sql
SELECT * FROM knowledge_base
WHERE tags CONTAINS 'billing'
REASON 'Do you offer any discounts?'
```

## Step 5: Handle Question Variations

The LLM understands different ways of asking the same question:

```sql
-- Formal
SELECT * FROM knowledge_base REASON 'What is the process for password recovery?'

-- Casual
SELECT * FROM knowledge_base REASON 'forgot my password help'

-- Specific
SELECT * FROM knowledge_base REASON 'I cannot log in because I do not remember my password'
```

All three should return similar answers about password reset.

## Step 6: Building a Q&A API

Here's how you might integrate ReasonDB into a chatbot or help widget:

```python
import requests

def ask_question(question, category=None):
    """Ask a question to the knowledge base."""

    # Build the query
    if category:
        query = f"SELECT * FROM knowledge_base WHERE metadata.category = '{category}' REASON '{question}'"
    else:
        query = f"SELECT * FROM knowledge_base REASON '{question}'"

    # Execute query
    response = requests.post(
        "http://localhost:4444/v1/query",
        json={"query": query}
    )

    result = response.json()

    if result["documents"]:
        doc = result["documents"][0]
        return {
            "answer": doc["answer"],
            "confidence": doc["confidence"],
            "source": doc["title"]
        }
    else:
        return {
            "answer": "I couldn't find an answer to your question.",
            "confidence": 0,
            "source": None
        }

# Usage
result = ask_question("How do I reset my password?")
print(f"Answer: {result['answer']}")
print(f"Confidence: {result['confidence']}")
print(f"Source: {result['source']}")
```

## Step 7: Improving Answer Quality

### Combine with Keyword Pre-filtering

For faster responses, use SEARCH to pre-filter, then REASON:

```sql
SELECT * FROM knowledge_base
SEARCH 'password reset'
REASON 'How do I reset my password?'
```

### Use Higher Confidence Threshold

Only return high-confidence answers:

```bash
# In your application code, filter by confidence
result = ask_question("...")
if result["confidence"] > 0.8:
    return result["answer"]
else:
    return "I'm not sure. Let me connect you with support."
```

## Best Practices

<Tip>
**Structure your content well**: Use clear headings (H1, H2, H3) to create a navigable tree. This helps the LLM find specific sections.
</Tip>

<Note>
**Tag by topic and difficulty**: Use metadata like `category` and `difficulty` to enable filtered searches for different user types.
</Note>

<Warning>
**Monitor confidence scores**: Low confidence answers may indicate gaps in your documentation. Track these to improve your knowledge base.
</Warning>
