# Backup & Recovery

ReasonDB provides comprehensive backup and recovery capabilities to protect your data. This guide covers full and incremental backups, point-in-time recovery, and data export/import.

## Overview

The backup system supports:

- **Full Backups** - Complete database snapshots with compression
- **Incremental Backups** - Only changes since the last backup
- **Point-in-Time Recovery** - Restore to any backup point
- **Export/Import** - JSON, JSONL, and CSV formats
- **Integrity Verification** - MD5 checksums for all backups

## CLI Commands

### Creating Backups

Create a full backup:

```bash
# Create a full backup
reasondb backup create --database data/reasondb.redb --backup-dir backups

# With description
reasondb backup create -d data/reasondb.redb -b backups -D "Before migration"

# Incremental backup
reasondb backup create -d data/reasondb.redb -b backups -t incremental
```

### Listing Backups

```bash
# List all backups
reasondb backup list -b backups

# JSON output
reasondb backup list -b backups --format json
```

Output:

```
Backups:

  snap_a1b2c3d4e5f6 [FULL] 2026-01-30 10:30:00 - 5 tables, 100 docs (2.5 MB)
    Before migration
  snap_inc_g7h8i9j0 [INCR] 2026-01-30 14:00:00 - 5 tables, 105 docs (512 KB)
```

### Restoring Backups

```bash
# Restore to a new database
reasondb backup restore snap_a1b2c3d4e5f6 --target restored.redb

# Overwrite existing database
reasondb backup restore snap_a1b2c3d4e5f6 --target data/reasondb.redb --force
```

### Verifying Backups

```bash
# Verify backup integrity
reasondb backup verify snap_a1b2c3d4e5f6

# Output: ✓ Backup snap_a1b2c3d4e5f6 is valid.
```

### Deleting Backups

```bash
# Delete with confirmation
reasondb backup delete snap_a1b2c3d4e5f6

# Skip confirmation
reasondb backup delete snap_a1b2c3d4e5f6 --yes
```

### Pruning Old Backups

```bash
# Keep last 5 full and 10 incremental backups
reasondb backup prune --keep-full 5 --keep-incremental 10

# Output: ✓ Pruned 3 backup(s)
```

## Export & Import

### Exporting Data

Export your data in various formats:

```bash
# Export everything to JSON
reasondb backup export data_export.json --database data/reasondb.redb

# Export specific table to CSV
reasondb backup export contracts.csv -d data/reasondb.redb --format csv --table legal

# Export to JSON Lines (streaming format)
reasondb backup export documents.jsonl -d data/reasondb.redb --format jsonl

# Include nodes in export
reasondb backup export full_export.json -d data/reasondb.redb --include-nodes
```

### Importing Data

```bash
# Import from JSON
reasondb backup import data_export.json --database data/reasondb.redb

# Import CSV into specific table
reasondb backup import contracts.csv -d data/reasondb.redb --format csv --table legal

# Import JSONL
reasondb backup import documents.jsonl -d data/reasondb.redb --format jsonl --table docs

# Update existing records instead of skipping
reasondb backup import data.json -d data/reasondb.redb --update
```

## Backup Strategy

### Recommended Schedule

| Backup Type | Frequency | Retention |
|-------------|-----------|-----------|
| Full | Weekly | 4 weeks |
| Incremental | Daily | 7 days |
| Before Migrations | On-demand | Until verified |

### Automated Backups

Set up automated backups with cron:

```bash
# Daily incremental at 2 AM
0 2 * * * /usr/local/bin/reasondb backup create -t incremental -D "Daily auto"

# Weekly full on Sunday at 3 AM
0 3 * * 0 /usr/local/bin/reasondb backup create -t full -D "Weekly auto"

# Prune old backups weekly
0 4 * * 0 /usr/local/bin/reasondb backup prune --keep-full 4 --keep-incremental 7
```

### Disaster Recovery Checklist

1. **Regular Testing** - Periodically test restores to a separate location
2. **Off-site Copies** - Store backups in a different location/cloud
3. **Verification** - Run `backup verify` after each backup
4. **Documentation** - Keep backup locations and recovery procedures documented
5. **Monitoring** - Alert on backup failures

One document per line for streaming processing:

```jsonl

```


## Programmatic Usage

### Rust

```rust
use reasondb_core::{
    BackupManager, BackupOptions, BackupType,
    ExportFormat, ExportOptions, Exporter,
    NodeStore,
};

fn main() -> anyhow::Result<()> {
    let store = NodeStore::open("data/reasondb.redb")?;
    
    // Create backup manager
    let manager = BackupManager::new(&store, "backups")?;
    
    // Create full backup
    let backup = manager.create_backup(&store, BackupOptions::full())?;
    println!("Created backup: {}", backup.id);
    
    // List backups
    for backup in manager.list_backups()? {
        println!("{}: {} docs", backup.id, backup.document_count);
    }
    
    // Export to JSON
    Exporter::export(&store, "export.json", ExportOptions::json())?;
    
    Ok(())
}
```

## Troubleshooting

### Checksum Mismatch

If you see "Checksum mismatch" errors:

1. The backup file may be corrupted
2. Try downloading/copying the backup again
3. Check disk health with `fsck`

### Database Lock Errors

If restore fails with lock errors:

1. Stop any running ReasonDB server
2. Ensure no other process is using the database
3. Try with `--force` to overwrite

### Import Conflicts

If imports skip many records:

1. Use `--update` flag to update existing records
2. Check for duplicate IDs in the import file
3. Verify the target table exists for CSV/JSONL imports
